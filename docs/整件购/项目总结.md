# 整件购项目总结

## 一、项目定位

1药城平台上线"整件购"专区，让客户按商品的**整件倍数**下单，提升仓储发货效率、降低拆零成本，同时满足批发客户对整件批量采购的需求。

**核心定位：高价值、窄人群、强运营的专项服务，而非通用功能。**

## 二、数据洞察（为什么是"窄人群"）

基于某周全平台约100万行采购数据分析：

| 维度 | 数据 |
|------|------|
| 整件采购行为占比 | 仅 **3%** 的采购行为满足整件数量 |
| 连锁客户整件占比 | 约 6% |
| 批发客户整件占比 | 约 **18%**（主力人群） |
| 曾被整件采购的品种 | 7,714 个 |
| 被 ≥5 个客户整件采购的品种 | 1,547 个 |
| 被 ≥10 个客户整件采购的品种 | 584 个 |
| 被 ≥20 个客户整件采购的品种 | 仅 **154** 个 |

**核心结论：**
1. 整件采购是低频需求，97%用户不按整件购买
2. 批发客户是主力（18%），远高于连锁（6%）和单体
3. 高热度整件商品极少，真正被多人反复整件采购的不足200个
4. 选品必须高度精准，不能靠"开放设置"自然形成

## 三、两种互补模式

### 1. 预售整件购（无实物库存，以销定采）

| 要素 | 说明 |
|------|------|
| 适用场景 | 供应链预判有整件需求但暂无库存的商品 |
| 运作机制 | 供应链圈定候选商品 → 用户下单 → 系统生成采购需求 → "先订后采，零压货" |
| 商家参与 | JBP商家通过后台"活动报名"主动加入，设置预售价格与最大可售件数 |
| 用户体验 | 独立专区展示 + 主搜带"整件预售"标签 + **独立购物车**（避免与现货混淆） |
| 系统模式 | JBP 1.0流程，客户下单后再下PO单 |

### 2. 实物整件购（有现货库存）

| 要素 | 说明 |
|------|------|
| 适用场景 | 已有整件库存且历史数据显示存在整件采购行为的商品 |
| 运作机制 | JBP商家/自营采购将采购倍数改为"件包装" → 系统每日自动聚合至专区 |
| 用户体验 | 常规购物车 + 标准下单流程 |
| 系统模式 | JBP 3.0可快速支持 |

### 双向导流
- 预售专区 → 提供入口跳转至"实物整件购"
- 实物专区 → 提供入口跳转至"预售整件购"
- 目标：让用户在"整件采购"心智下无缝发现所有可整件购买的商品

## 四、业务规则

| 规则项 | 说明 |
|------|------|
| 收佣标准 | 1.5% |
| 操作费 | 当前免操作费、免环线 |
| 价格构成 | 公开价，偏远地区加价不超过0.5% |
| 商品范围 | 药品类，全品 |
| 仓库范围 | 自营四仓 |
| 发货时效 | 属地区域2天，非属地区域4天，支持采购设置 |
| 是否进主搜 | 是 |
| 是否有专区 | 是，专区内支持搜索 |
| 平台补贴活动 | 参加 |
| 退货 | 支持拆零退货，质量/破损走小额理赔和报损流程 |

## 五、项目目标（3月）

| 维度 | 3月目标 |
|------|--------|
| GMV规模 | **1600W** |
| 参与SKU数 | 待拆解（预售+实物） |

目标拆解为预售（JBP1.0 + JBP3.0）和实物两个版本，关注N700核心品种的库存≥100 SKU覆盖。

## 六、已报名情况

目前已有多家JBP商家报名参与，覆盖品种包括：
- **慢病用药：** 硝苯地平缓释片、贝那普利片、非布司他片等
- **抗感染：** 头孢克肟分散片、阿奇霉素片、磷酸奥司他韦胶囊等
- **OTC常用药：** 感冒灵颗粒、感康、健胃消食片、连花清瘟颗粒等
- **外用药：** 湿润烧伤膏、硝酸咪康唑乳膏等

商家分布涵盖广东、湖北、重庆、四川、陕西、安徽等多个大区。

## 七、问题与风险

| 问题/风险 | 说明 |
|----------|------|
| **预售专区搜索能力冗余** | 商品数量极少（预计<200个活跃SKU），接入完整搜索系统ROI极低 |
| **实物整件购可能降低销售份额** | 大部分需求是非整件购买，若商家自主设置"仅整件"会丧失散货销售机会 |
| **运营要求高** | 小场景需求需要精细化运营选品+定价，否则很难有效果 |
| **搜索能力不足** | 当前仅支持通用名模糊匹配，不支持规格/厂家等B端关键筛选 |
| **实物专区自动聚合** | 需额外开发工作量 |
| **双专区跳转** | 实物→预售的跳转暂不支持，待开发 |
