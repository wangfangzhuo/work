# 1药城前台价格体系：加价逻辑全解

> **整理时间**：2026-03-25
> **信息来源**：内部录音整理

---

## 一、三种业务模式

| 简称 | 全称 | 说明 |
|------|------|------|
| **自营** | 自营备货 | 平台自采自销 |
| **聚宝盆（JBP）** | 自营聚宝盆 | 商家供货，平台代销 |
| **MP** | 商家入驻 | 第三方商家自主经营 |

不同业务模式的加价逻辑有所不同（详见下文）。

---

## 二、基础价格的确定（三级取价）

每个商品只有一个基础价格，按以下优先级从高到低取值：

```
PIS条件价  >  人工条件价  >  公开价
```

| 价格层级 | 说明 |
|----------|------|
| **放贷底价（成本价）** | 系统中的底层成本价，每个商品一个 |
| **公开价** | 采购在底价基础上设定，每个商品一个 |
| **人工条件价** | 在公开价基础上，按**区域（省份）× 客户类型（单体/连锁/批发等）** 人工设置不同价格或涨降比例 |
| **PIS条件价** | 由PIS系统自动设置，托管商品自动生成条件价，**优先级最高**（高于人工条件价和公开价） |

---

## 三、加价链路（四层叠加）

在基础价格确定后，用户最终看到的前台价格 = 基础价格经过以下**四层加价**依次叠加：

```
基础价格（PIS条件价 / 人工条件价 / 公开价）
    │
    ▼ 第①层
环线加成（按区域+客户类型加点）
    │
    ▼ 第②层
操作费（仅聚宝盆）
    │
    ▼ 第③层
客户标签加价
    │
    ▼ 第④层
增值标签加价
    │
    ▼
前台展示价格
```

---

### 第①层：环线加成

通过自营后台「统一加价设置」功能配置。**自营和聚宝盆规则略有不同**：

#### 自营备货的环线加成

| 设置方式 | 说明 |
|----------|------|
| **区域 × 客户类型 → 加点比例** | 不同区域、不同客户类型设置不同的加价百分比 |
| **商品黑名单** | 导入黑名单的商品，绕过环线加价 |
| **商品 × 客户黑名单** | 指定一批客户购买指定商品时，免环线加成 |

#### 聚宝盆的环线加成

| 设置方式 | 说明 |
|----------|------|
| **区域 × 客户类型 → 加点比例** | 同自营 |
| **商品黑名单** | 同自营 |
| ~~商品 × 客户黑名单~~ | **不支持**（与自营的区别） |

---

### 第②层：操作费

> **仅聚宝盆（JBP）有操作费，自营和MP均无。**

| 设置方式 | 说明 |
|----------|------|
| **基础规则** | 按商品公开价所处的**价格区间段**，定义不同的加价金额 |
| **白名单** | 指定品种设置额外的加价规则 |
| **黑名单** | 指定品种绕过操作费加价 |

---

### 第③层：客户标签加价

一个用户**只能属于一个客户标签**。

| 配置维度 | 说明 |
|----------|------|
| **商品范围** | 三选一：全量商品生效 / 排除部分商品 / 仅白名单商品生效 |
| **特价活动** | 可配置特价活动商品是否也叠加客户标签加价 |
| **聚宝盆** | 可配置聚宝盆商品是否也叠加客户标签加价 |

---

### 第④层：增值标签加价

一个用户**只能属于一个增值标签**（但可以同时有一个客户标签 + 一个增值标签 → **两层加价叠加**）。

| 配置维度 | 说明 |
|----------|------|
| **商品范围** | 三选一：全量商品生效 / 排除部分商品 / 仅白名单商品生效 |
| **特价活动** | 可配置特价活动商品是否叠加增值标签加价 |
| **聚宝盆** | 可配置聚宝盆商品是否叠加增值标签加价 |

---

## 五、活动价格逻辑

商品参加活动时，加价链路会被部分绕过。不同活动类型的规则如下：

### 特价活动（特价 / 特惠价）

特价活动在基础价格上设定活动价，但**客户标签加价和增值标签加价仍然生效**。区别在于环线加成和操作费是否绕过：

| 活动类型 | 环线加成 | 操作费 | 客户标签加价 | 增值标签加价 |
|----------|:--------:|:------:|:----------:|:----------:|
| **特价** | ❌ 绕过 | ❌ 绕过 | ✅ 生效 | ✅ 生效 |
| **特惠价** | ❌ 绕过 | ✅ 生效 | ✅ 生效 | ✅ 生效 |

> 特价 = 绕过环线 + 操作费；特惠价 = 仅绕过环线，操作费仍计算。

### 秒杀

**绕过所有加价层级**，活动价直接作为前台展示价格。

| 加价层级 | 是否生效 |
|----------|:--------:|
| 环线加成 | ❌ |
| 操作费 | ❌ |
| 客户标签加价 | ❌ |
| 增值标签加价 | ❌ |

> **注**：秒杀仅自营和聚宝盆有，MP没有秒杀。

### 拼团 / 当品包邮

与上述活动不同，拼团和当品包邮会在前台**生成一个全新的独立商品卡片**，与原普通商品卡片互不影响，属于独立的商品展示逻辑。

## 六、完整价格计算总览

```
                    放贷底价（成本价）
                         │
                    ┌────┴────┐
                    │  采购定价  │
                    └────┬────┘
                         ▼
              ┌─ PIS条件价（最高优先级）
  基础价格 ◄──┼─ 人工条件价
              └─ 公开价（兜底）
                         │
         ┌───────────────┼───────────────┐
         │               │               │
      普通商品        特价活动          秒杀
         │               │               │
    ①环线加成      特价：跳过①②      全部跳过
         │         特惠：跳过①        直接展示
    ②操作费(JBP)        │
         │          ③客户标签
    ③客户标签            │
         │          ④增值标签
    ④增值标签            │
         │               │
         ▼               ▼
      前台展示价格     前台展示价格

  * 拼团/当品包邮 → 生成独立商品卡片，不走上述加价链路
```
