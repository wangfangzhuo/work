# Agent 下医药入口形态讨论

## 核心判断

| 传统入口 | Agent 入口 |
|---------|-----------|
| 抢**注意力** — 用户来了，自己选 | 抢**决策权** — 用户不看了，Agent 替他选 |

> **谁拿到决策权，谁决定钱往哪流。**
> 讨论入口形态的本质：我们能不能成为药店经营决策的代理人。

---

## 决策权的变现空间

| 模式 | 收费对象 | 逻辑 |
|------|---------|------|
| 订单流分发 | 供应商平台 | 竞价佣金 |
| 新品推广 | 药企 | 精准触达采购决策者 |
| 省钱分成 | 药店 | 帮你省5万，收你1万 |
| 集采议价 | 供应商 | 聚合千家药店采购量 |
| 数据风控 | 金融机构 | 最真实的经营数据 |

前提都一样 —— **先把决策权拿到手**。

---

## 信任三步走

```
第一步                  第二步                  第三步
让他看见在亏钱    →    反复验证建信任    →    接管日常决策

· 算一笔账             · Agent 给建议          · 常规决策自动执行
· 给出扎心数字         · 老板自己验证          · 异常才打扰老板
· 制造"我需要"         · 采纳率 >70% = 信了    · 决策权正式转移
```

关键问题：**第一步那个钩子，用什么场景来扎？**

---

## 第一刀切哪里

**五个判断标准**：痛感强 / 价值可量化 / 验证门槛低 / 交互频率高 / 数据可延伸

| | 方案A：跨平台比价 | 方案B：效期损耗 | 方案C：组合打法 |
|---|---|---|---|
| **钩子** | "你上月多花了¥3,200" | "47个品种正在变废品，货值¥2.3万" | 效期做钩子，比价做留存 |
| **痛感** | ★★★★ 可省的钱 | ★★★★★ 正在亏的钱 | ★★★★★ |
| **量化** | ★★★★★ | ★★★★★ | ★★★★★ |
| **门槛** | ★★★ 需授权平台账号 | ★★★ 需库存数据 | ★★★ |
| **频率** | ★★★★ 每天采购 | ★★★ 每周关注 | ★★★★ |
| **延伸** | ★★★★ 采购→库存→动销 | ★★★★★ 库存→补货→采购 | ★★★★★ |
| **风险** | 爬虫法律风险，平台反制 | 合规风险小 | 需两套能力 |

**组合打法路径**：效期诊断炸醒老板 → 补货需求 → 自然接入比价 → 日常信任 → 接管采购

> **今天重点讨论：第一刀到底切哪个场景？**

## 终局：采购代理 → 经营代理

```
采购决策 → 库存管理 → 定价决策 → 品类选品 → 经营全局
从哪买     效期/补货   参考同行   需求分析   帮你把店
买多少     周转优化   利润分析   新品推荐   开好
```

采购数据 → 库存数据 → 销售数据 → 经营全貌（同一个 Agent 的自然延伸）

---

## 三层壁垒

| 壁垒 | 内容 | 为什么抄不走 |
|------|------|-------------|
| **数据** | 跨平台价格 + 药店经营数据 | 需要时间积累，后来者晚一年 |
| **关系** | 供应商偏好、采购习惯、经营特征 | 沉淀在 Agent 记忆里，迁移成本极高 |
| **网络** | 药店越多→议价越强→省钱越多→更多药店 | 飞轮效应，工具型竞品接不住 |

> **工具可以被抄，决策数据和用户信任抄不走。**
