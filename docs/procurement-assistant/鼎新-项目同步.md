## 一、项目概况

**项目名称**：鼎新（pharma-compare）
**产品定位**：药品多平台比价 AI Agent，基于 openclaw 智能体基座，为药店采购人员提供对话式跨平台比价服务。

**业务背景**：
- 公司：1药城（111, Inc.），B2B 医药分销平台
- 痛点：药店采购人员需在药师帮、九洲通、199 等多个平台手动比价，繁琐低效
- 战略意图：抢夺医药采购"流量入口"，成为类似"什么值得买"的流量分发平台，通过极致透明的比价体验留住用户

**三重目标**：

| 目标 | 内容 |
|------|------|
| 业务目标 | 上线可用的单品多平台比价能力，**21 天内产生第一张真实订单**（目标：2026-04-03 前） |
| 组织目标 | 实践 Spec as Design 方法论，由产品和设计师完成从需求到代码的闭环 |
| 战略探索 | 验证 AI Agent 形态在 B2B 医药采购场景的可行性 |

---

## 二、产品形态

基于 **openclaw 智能体基座**构建，不是传统业务系统：

- **核心界面**：对话窗口，用户通过自然语言查价，不是表格筛选器
- **能力封装**：查价、下单等操作封装为 openclaw Skills（每个 Skill 是独立能力单元）
- **引导式操作**：Agent 主动引导用户完成授权、选平台、确认药品等步骤
- **目标用户**：单体药店老板、连锁药店采购负责人（非技术人员，会打字就能用）

## 四、V1 版本范围

### 做什么

- 对话式交互界面（Web），基于 openclaw
- 用户平台账号授权（至少 1 个平台）
- 单品搜索 + 多平台价格查询（封装为查价 Skill）
- 比价结果结构化卡片展示（各平台价格 + 差价 + 最低价标注）
- 搜索拦截与导流入口方案
- 辅助下单能力（模拟登录、加车，封装为下单 Skill）

### 不做什么（V2+）

- Agent 主观采购建议（需行业数据积累）
- 模糊匹配比价（V1 只做精确匹配：同名+同规格+同厂家）
- 批量比价 / 采购清单管理
- 去中心化爬虫（V1 先中心化）

### V1 上线标准

- 至少 1 个平台（建议药师帮）可稳定完成：登录 → 搜品 → 返回价格
- 具备辅助下单能力（加车流程可跑通）
- 21 天内产生第一张真实订单

## 九、技术路线决策（启动会，2026-03-13）

- **爬虫方案**：先**中心化**（服务器端爬虫）快速跑通，后续视封 IP 风险再考虑去中心化
- **中心化风险**：容易被平台封锁 IP，成本约 1元/IP/天
- **去中心化优势**：利用客户本地 IP 绕过反爬，但环境复杂
