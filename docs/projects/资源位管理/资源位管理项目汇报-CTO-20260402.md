## 二、开发做了什么（技能包底座）

开发同学（叶金钢）完成了技能包的底座，对接了 OpenClaw 平台和后台接口。

### 技能包结构

```
pc-home-carousel/
├── skill.json    # 技能元信息（名称、版本、触发入口、MCP 协议声明）,用于告诉 OpenClaw 平台这个技能叫什么、怎么启动
├── package.json  # Node.js 包配置（依赖 MCP SDK、axios）
├── index.js      # 核心逻辑（接口调用），定义工具的入参（声明 7 个参数（楼层名称、分站编码、图片地址、跳转链接、开始时间、结束时间、企业类型），均为必填）+调用后台的接口（POST 到资源位创建接口，携带操作者身份（从 OpenClaw session 中读取），返回创建结果）
└── skill.md      # 行为规则文件（控制 AI 行为的规则文件，OpenClaw 读取后注入给大模型），初版仅有基础描述，由 PM 侧补充业务规则（见下节）
```

### 各文件职责

**skill.json** — 告诉 OpenClaw 平台这个技能叫什么、怎么启动：
```json
{
  "name": "PC首页轮播图管理",
  "code": "pc-home-carousel",
  "type": "mcp",
  "mcp": { "command": "node", "args": ["./index.js"] }
}
```

**index.js** — 技能的可执行部分，做了三件事：
1. **定义工具入参**：声明 7 个参数（楼层名称、分站编码、图片地址、跳转链接、开始时间、结束时间、企业类型），均为必填
2. **企业类型映射**：内置中文名称 → 接口编码的对照表（如「单体药店」→ `3*单体药店`）
3. **调用后台接口**：POST 到资源位创建接口，携带操作者身份（从 OpenClaw session 中读取），返回创建结果

**skill.md** — 控制 AI 行为的规则文件，OpenClaw 读取后注入给大模型。初版仅有基础描述，由 PM 侧补充业务规则（见下节）。

### 技术选型

| 组件 | 说明 |
|------|------|
| MCP 协议 | OpenClaw 平台的技能接入标准，stdio 模式本地运行 |
| Node.js + axios | 轻量级，快速对接现有后台 HTTP 接口 |
| skill.md | 控制 AI 行为逻辑的规则层，PM 可独立维护 |

### v1.1 更新：操作者身份识别与日志

## 三、产品做了什么

### 问题

开发交付的技能包，skill.md **只有基础描述，没有业务规则**。运营必须把所有技术参数一字不差提供，才能触发创建，和填表单没有本质区别：

```
图片地址：https://p8.maiyaole.com/xxx.jpg
跳转链接：https://mall.yaoex.com/
开始时间：2026-05-01 00:00:00
结束时间：2026-05-05 23:59:59
企业类型：1,2,3,4,5,8,71
楼层名称：汤臣倍健五一活动
分站编码：000000
```
任何一项遗漏或格式不对，配置均无法完成。

### 做了什么

在 skill.md 中补充了三层业务规则：

**① 参数智能补全**

| 字段 | 原来 | 现在 |
|------|------|------|
| 分站编码 | 必须输入 `000000` | 未提及时默认「默认分站」 |
| 楼层名称 | 必须自己起名 | 未提及时 AI 根据活动内容自动生成（10字以内） |
| 企业类型 | 必须输入数字编码如 `3,4,8` | 说「除纯批发」「连锁」「全部」，AI 自动映射对应编码 |
| 生效时间 | 必须写 `2026-05-01 00:00:00` 格式 | 说「五一法定假期那五天」，自动解析精确时间段 |

**② 人机确认环节**

调用接口前，AI 将配置项以用户友好格式列出，让运营确认无误后再提交，内部编码不暴露给用户：

```
📋 轮播图配置清单
| 楼层名称 | 汤臣倍健五一特惠                                         |
| 生效周期 | 2026-05-01 ~ 2026-05-05                                 |
| 可见企业 | 单体药店、公立医疗机构、非公立医疗机构、个体诊所、连锁特许经营 |
| 跳转链接 | https://mall.yaoex.com/                                  |
```

**③ 业务规则版本化沉淀**

企业类型映射表、时间解析规则、默认值逻辑全部写入 skill.md，**成为可版本管理的团队资产**，后续调整规则不依赖开发。

### 效果

运营只需说：
> 「马上要五一了，汤臣倍健厂家给了钱让我们再加一个轮播图」

AI 主动追问图片地址和跳转链接两个必填项，其余全部自动处理：

✅ 汤臣倍健五一主题轮播图已成功创建
- 轮播图 ID：1720498
- 生效时间：2026-05-01 00:00:00 至 2026-05-05 23:59:59
- 可见范围：非批发类型全部企业

**全程 3 轮对话，约 2 分钟完成。**

## 四、⚠️ 当前安全风险与待办

### 风险

当前技能包**没有权限校验**。任何获得该技能包的人，都可以直接操作 PC 首页配置，无需审批或身份验证。一旦技能包被安装到不受控账号，内部系统将面临未授权操作风险。

### 应有的调用链

```
用户发起对话
    ↓
从 session 获取 currentUserId  ← 已完成
    ↓
调用【权限校验 MCP 服务】       ← 待 IT 支持
    ├── ✅ 有权限 → 展示配置清单 → 用户确认 → 调用创建接口
    └── ❌ 无权限 → 「您没有该操作权限，请联系管理员申请」，终止
```

### 待 IT 团队提供

一个权限校验 MCP 服务，入参：

| 参数 | 说明 |
|------|------|
| `userId` | 操作人 ID（技能包已可提供） |
| `action` | 操作类型（如 `pc_carousel_create`） |

返回：`{ "allowed": true/false, "reason": "..." }`

IT 服务就绪后，开发在 index.js 加入调用即可闭环。**此前技能包不应分发给非受控账号。**

## 六、核心判断

这个项目验证了一件事：**AI 落地的关键不是接口，是业务规则层**。

开发打通了系统连接，但没有业务规则的 AI 只是一个严格的表单。PM 用 skill.md 把「运营的经验和判断」编码进去后，才真正让自然语言操作成为可能。

后续迭代主要由 PM 侧驱动：补充规则、优化话术、扩展资源位类型，**不需要每次都拉开发**。
