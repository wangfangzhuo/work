# 资源位管理 AI 化项目进展汇报

> **日期**：2026-04-02
> **汇报对象**：CTO
> **项目阶段**：一期 Demo 完成

---

## 一、做了什么

**目标**：让运营通过自然语言在 OpenClaw 上完成资源位创建，不再需要手动登录后台、逐字段填写表单。

**一期范围**：PC 首页轮播图的新增配置，通过对话完成，全程不切换系统。

---

## 二、我做了什么（PM 侧）

开发同学（叶金钢）提供了技能包的底座：连通了接口、定义了触发词，但 **skill.md 只有空壳**，没有业务规则，运营必须把所有技术参数一字不差地说出来才能触发创建。

我在此基础上补充了 skill.md 的业务规则层，具体做了三件事：

### 1. 参数智能补全（让运营说人话，不说接口参数）

| 场景 | 原来 | 现在 |
|------|------|------|
| 分站编码 | 必须输入 `000000` | 未提及默认填「默认分站」 |
| 楼层名称 | 必须自己起名 | 未提及时 AI 根据活动内容自动生成（10字以内） |
| 企业类型 | 必须输入数字编码（如 `3,4,8`） | 说「除纯批发」「连锁」「全部」即可，AI 自动映射到对应编码 |
| 生效时间 | 必须写 `2026-05-01 00:00:00` 格式 | 说「五一法定假期那五天」，AI 自动解析为精确时间段 |

### 2. 人机确认环节（防错）

调用接口前，AI 将所有配置项用**用户友好的格式**列出，让运营确认无误后再提交。内部映射的数字编码不对用户暴露，只展示中文业务语言。

```
📋 轮播图配置清单
| 配置项     | 内容                              |
| 楼层名称   | 汤臣倍健五一特惠                    |
| 生效周期   | 2026-05-01 ~ 2026-05-05 五天      |
| 可见企业   | 非批发全部类型（7种）               |
| 跳转链接   | https://mall.yaoex.com/          |
```

### 3. 完整业务规则沉淀

将企业类型映射表、时间解析规则、默认值逻辑全部写入 skill.md，**业务规则成为可版本管理的资产**，后续调整不依赖开发。

---

## 三、效果对比

### 改造前（开发交付的原始版本）

运营必须提供所有技术参数才能触发：
```
图片地址：https://p8.maiyaole.com/xxx.jpg
跳转链接：https://mall.yaoex.com/
开始时间：2026-05-01 00:00:00
结束时间：2026-05-05 23:59:59
企业类型：1,2,3,4,5,8,71
楼层名称：汤臣倍健五一活动
分站编码：000000
```

**任何一项遗漏或格式不对，配置均无法完成。**

### 改造后（加入业务规则后）

运营只需说：
> 「马上要五一了，汤臣倍健厂家给了钱让我们再加一个轮播图」

AI 主动追问两个必填项（图片地址、跳转链接），其余全部自动处理，最终：

✅ 汤臣倍健五一主题轮播图已成功创建
- 轮播图 ID：1720498
- 生效时间：2026-05-01 00:00:00 至 2026-05-05 23:59:59
- 可见范围：非批发类型全部企业

**全程 3 轮对话，约 2 分钟完成。**

## 五、核心判断

这个项目验证了一件事：**AI 落地的关键不是接口，是业务规则层**。

开发同学打通了系统连接（1 周），但没有业务规则的 AI 只是一个严格的表单。PM 用 skill.md 把「运营的经验和判断」编码进去后，才真正让自然语言操作成为可能。

这也意味着后续的迭代主要由 PM 侧驱动：补充规则、优化话术、扩展资源位类型，**不需要每次都拉开发**。
