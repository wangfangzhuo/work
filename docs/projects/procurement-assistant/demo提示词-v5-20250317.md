## 变更说明（v4 → v5）

| 维度 | v4 | v5（本版） |
|------|-----|-----------|
| 欢迎页 | 3条文字消息（欢迎语+能力介绍+引导配置） | **HeroIntro 组件**：标题+3个能力卡片+黄色授权提示框 |
| 消歧时机 | 搜索前「规格确认」（基于历史偏好推荐规格） | **爬取后「数据汇总」**（按厂家+规格聚合真实数据，用户选择） |
| 消歧卡片 | SpecConfirmation（推荐规格+可选列表+确认按钮） | **DataSummary**：厂家×规格候选卡片，每项含 HighlightBadge，**点击即确认**（无单独确认按钮） |
| 候选标签 | 无 | 每个候选带 HighlightBadge：📘历史采购（蓝色）/ 📈市场最畅销（橙色）/ 💰性价比最高（绿色） |
| 供应商管理 | 仅内联表单 SupplierForm | 新增 **AddSupplierModal**（添加弹窗）+ **SupplierConfigModal**（配置弹窗，含授权管理+报价单管理） |
| 架构定位 | 未明确区分 | **Skill 只负责 AI 智能层**，数据爬取由工程化服务完成 |

## 关键词触发规则汇总

| 输入关键词 | 触发场景 | 异常平台 | 特殊交互 |
|-----------|---------|---------|---------|
| 感冒灵（或含「感冒」） | 正常场景 | 无 | 采集→汇总3种候选→用户选→比价 |
| 阿莫西林（或含「莫西」） | 授权过期场景 | 小药药 expired | AuthDecision 二选一→汇总3种候选→比价 |
| 感康 | 数据获取失败场景 | 药师帮 error | hover tooltip→汇总2种候选→比价 |
| 结算/购物车/汇总 | 采购计划场景 | — | 清单汇总 + 结算 |
| 其他任意输入 | 走感冒灵正常流程 | 无 | 用输入的词替换药品名 |
