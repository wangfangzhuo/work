# 鼎新 Skills 清单（整合版）

> 记录时间：2026-03-17
> 基于：用户旅程5阶段 + V1 Scope + 代码Demo交互 + 团队讨论
> 架构：openclaw 智能体基座 + Skills
> 设计原则：数据获取层（登录、爬取、归档）由工程化代码实现，Skill 只负责 AI 智能层

---

## 架构分层

```
┌─────────────────────────────────────────────┐
│  Skill（AI 智能层）                          │
│  理解意图 → 汇总归类 → 引导消歧 → 比价展示   │
├─────────────────────────────────────────────┤
│  工程化服务（后端）                           │
│  账号管理 / 平台登录 / 数据爬取 / 数据归档     │
└─────────────────────────────────────────────┘
```

**Skill 不处理**：用户账号密码的输入与存储、各平台登录认证、数据爬取与归档
**Skill 负责**：意图识别、数据汇总归类、消歧引导、比价分析、采购决策辅助

## Skill 定义：b2b_procurement_compare（B2B跨平台采购比价）

### 基本信息

| 字段 | 值 |
|------|-----|
| 名称 | b2b_procurement_compare |
| 触发条件 | 用户搜索药品 / 询问价格 / 要求比价 |
| V1 优先级 | P0 |

### 完整执行流程

```
用户输入通用名（如"阿莫西林"）
    │
    ▼
[模块1] 搜索拦截与引导
    │  判断是否展示比价入口（30天拒绝记忆）
    │
    ▼ 用户点击"立即比价"
[工程化服务] 授权状态检查
    │  检查各平台登录态是否有效
    │  ├── expired → 引导用户重新授权（或跳过该平台）
    │  └── active → 继续
    │
    ▼
[工程化服务] 多平台并行爬取
    │  各平台搜索该通用名，返回所有匹配结果
    │  （N条结果 = 多厂家 × 多规格 × 多平台）
    │
    ▼
[模块2] 数据汇总与消歧
    │  将爬回的原始数据按「厂家+规格」聚合归类
    │  ├── 仅1种组合 → 跳过消歧，直接比价
    │  └── 多种组合 → 展示选项卡片，让用户选择
    │
    ▼ 用户确认目标药品（厂家+规格）
[模块3] 跨平台比价分析
    │  筛选该厂家+规格在各平台的价格
    │  按实际支付价排序，标注最低价
    │  标注库存、效期、优惠、配送时效
    │  异常平台（expired/error/timeout）单独分组
    │
    ▼
[模块4] 采购决策辅助
    │  ├── 线上平台 → 辅助加入1药城购物车
    │  └── 导出需求 → 生成采购计划（Excel）
    │
    ▼
    完成
```

---

### 模块1：搜索拦截与引导

**职责**：判断是否向用户展示跨平台比价入口

**输入**：
```typescript
{
  userId: string;          // 用户ID
  searchQuery: string;     // 用户搜索词
}
```

**输出**：
```typescript
{
  shouldShow: boolean;     // 是否展示引导
  reason?: 'never_rejected' | 'rejection_expired' | 'rejected_within_30d';
}
```

**规则**：
- 查询用户30天内是否明确拒绝过
- 已拒绝且未过期 → shouldShow: false
- 未拒绝 / 拒绝已过30天 → shouldShow: true
- 用户点击「不需要」时写入拒绝时间戳

---

### 模块2：数据汇总与消歧

**职责**：将各平台爬回的原始数据汇总归类，引导用户确认目标药品

**核心逻辑**：

用户输入的是通用名（如"阿莫西林"），各平台会返回大量结果，包含不同厂家、不同规格的药品。Skill 需要：

1. **聚合归类**：按「厂家+规格」维度去重聚合
2. **展示候选**：列出所有不同的「厂家+规格」组合，标注各组合覆盖了哪些平台
3. **用户选择**：让用户明确选择要比价的是哪个厂家的哪个规格

**输入**（来自工程化服务的爬取结果）：
```typescript
{
  query: string;           // 用户原始输入
  rawResults: Array<{      // 各平台爬回的原始数据
    platform: string;
    items: Array<{
      name: string;        // 药品名称
      genericName: string; // 通用名
      spec: string;        // 规格（如"0.25g×50粒"）
      manufacturer: string;// 厂家
      price: number;       // 单价
      stock: number;       // 库存
      expiry: string;      // 效期
      promotion?: string;  // 优惠信息
      seller?: string;     // 商家名称
    }>;
    status: 'success' | 'expired' | 'error' | 'timeout';
    error?: string;
  }>;
}
```

**输出**：
```typescript
{
  // 聚合后的候选列表
  candidates: Array<{
    manufacturer: string;      // 厂家
    spec: string;              // 规格
    genericName: string;       // 通用名
    matchedPlatforms: string[];// 哪些平台有这个药
    priceRange: [number, number]; // 价格区间（最低~最高）
    platformCount: number;     // 覆盖平台数
  }>;
  needsUserConfirm: boolean;  // 是否需要用户确认
  // 异常平台信息
  abnormalPlatforms: Array<{
    platform: string;
    status: 'expired' | 'error' | 'timeout';
    reason: string;
  }>;
}
```

**规则**：
- 同名+同规格+同厂家 = 同一药品
- 仅1个候选 → needsUserConfirm: false，直接进入比价
- 多个候选 → needsUserConfirm: true，展示选项卡片
- 按覆盖平台数降序排列（覆盖越多越靠前）
- 展示每个候选的价格区间，帮助用户快速判断

---

### 模块3：跨平台比价分析

**职责**：对用户选定的目标药品（厂家+规格），展示跨平台价格对比

**输入**：
```typescript
{
  drug: {
    genericName: string;   // 通用名
    spec: string;          // 规格
    manufacturer: string;  // 厂家
  };
  // 从模块2的rawResults中筛选出匹配项
  matchedItems: Array<{
    platform: string;
    price: number;
    stock: number;
    expiry: string;
    promotion?: string;
    serviceRating?: number;
    deliveryTime?: string;
    seller?: string;
  }>;
}
```

**输出**：
```typescript
{
  comparison: Array<{
    platform: string;
    seller?: string;
    price: number;
    stock: number;
    expiry: string;
    promotion?: string;
    serviceRating?: number;
    deliveryTime?: string;
    priceDiff?: number;    // 与最低价的差值
    isLowest: boolean;
  }>;
  lowestPrice: number;
  lowestPlatform: string;
  abnormalPlatforms: Array<{
    platform: string;
    status: 'expired' | 'error' | 'timeout';
    reason: string;
  }>;
  scrapedAt: Date;
}
```

**规则**：
- 按实际支付价格（含优惠后）排序
- 标注最低价平台
- 异常平台单独分组展示（授权过期、采集失败、超时）
- 标注数据查询时间
- V1 只展示客观数据，不做主观采购建议

---

### 模块4：采购决策辅助

**职责**：帮助用户完成采购动作

**能力**：
- **加入购物车**：将指定商品加入1药城购物车（V1 仅支持1药城自营）
- **生成采购计划**：将采购清单导出为 Excel 文件

**加车输入/输出**：
```typescript
// 输入
{
  platform: string;        // V1 仅 '1yc'
  drugId: string;
  quantity: number;
  userId: string;
}

// 输出
{
  success: boolean;
  error?: 'session_expired' | 'out_of_stock' | 'min_order_not_met';
  message?: string;
}
```

**采购计划输入/输出**：
```typescript
// 输入
{
  items: Array<{
    drugName: string;
    spec: string;
    manufacturer: string;
    platform: string;
    price: number;
    quantity: number;
  }>;
}

// 输出
{
  fileUrl: string;         // 下载链接
  fileName: string;        // 如"采购计划_20260317.xlsx"
  summary: {
    totalItems: number;
    totalAmount: number;
    platforms: string[];
  };
}
```

**规则**：
- V1 加车仅支持1药城，其他平台仅做价格展示
- 支付环节不在助手内完成，只辅助到加车
- 采购计划支持多药品汇总导出

## 工程化服务接口（Skill 调用的后端能力）

Skill 通过调用以下工程化接口获取数据，这些接口由后端团队实现：

| 接口 | 职责 | 负责人 |
|------|------|--------|
| `auth.checkStatus(userId, platforms)` | 检查各平台授权状态 | 林超/小白 |
| `auth.login(platform, credentials)` | 登录指定平台 | 林超/小白 |
| `scraper.search(query, platforms)` | 多平台并行爬取搜索结果 | 林超/小白 |
| `cart.add(platform, drugId, qty)` | 加入购物车 | 1药城团队 |
| `export.procurement(items)` | 生成采购计划Excel | 后端 |

## 扩展 Skill（P1/P2）

以下能力在 V1 视工期决定是否纳入，可作为独立模块加入主 Skill，或拆为独立 Skill：

### 规格推荐（P1）

在消歧阶段，基于用户历史采购记录推荐最可能的规格：
- 优先推荐历史采购最多的规格
- 标注市场最畅销规格
- V1 可简化为仅展示可选规格列表

### 解析线下报价单（P2）

解析用户上传的线下供应商报价单（Excel/PDF），提取药品价格数据纳入比价池：
- 支持 Excel (.xlsx, .xls) 和 PDF 格式
- AI 自动识别表格结构
- 解析后的数据参与跨平台比价

## 与之前 11-Skill 方案的对比

| 维度 | 之前（11个独立Skill） | 现在（1个主Skill + 工程化服务） |
|------|---------------------|-------------------------------|
| Skill 数量 | 11 个 | 1 个主 Skill（内含4个模块） |
| 数据获取 | Skill 内处理 | 工程化服务处理 |
| 上下文传递 | 跨 Skill 传参 | 模块间天然共享 |
| 消歧时机 | 搜索前（预设） | **爬取后（基于真实数据）** |
| 调试复杂度 | 需跟踪多个 Skill 调用链 | 单 Skill 内部排查 |
| 适合阶段 | 平台化、多场景复用 | **MVP / V1 快速交付** |

**关键变化**：消歧从「搜索前猜测」改为「爬取后基于真实数据汇总」，更符合实际业务流程——用户输入通用名，爬回一堆结果，Skill 汇总归类后让用户选。
