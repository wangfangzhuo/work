# 注册页面交互优化方案 v2

> **日期**：2026-03-27（v2 更新：基于产品细化方案）
> **背景**：业务反馈注册环节太复杂，用户需要手动填写大量与证照重复的信息
> **核心思路**：证照 OCR 自动填充 + 地址快捷复用 + 经营范围智能预填

---

## 一、改动模块总览

| 模块 | 改动点 |
|------|--------|
| 基本信息 | 新增证照上传入口，OCR 自动填充企业名称/法人/省市区/地址 |
| 仓库地址信息 | 新增身份证上传入口，OCR 填充收货人姓名；收货人手机号默认填注册手机号；收货地址/销售单打印地址增加【同注册地址一致】 |
| 经营范围 | 企业名称调用主数据接口自动预选；无返回值时按企业类型默认勾选（规则待补充） |
| 业务员子账号 | 字段更名，placeholder 更新 |

---

## 二、各模块详细设计

### 2.1 基本信息模块

#### 新增：证照上传入口

位置：在「基本信息」卡片顶部，字段表单上方。

```
┌──────────────────────────────────────────────────────────────────┐
│ 基本信息                                                          │
│                                                                  │
│  ┌──────────────────────────────────────────────────────────┐    │
│  │  上传证照可自动识别填写（选填）                              │    │
│  │                                                          │    │
│  │  ┌───────────────────────┐                               │    │
│  │  │   📄 营业执照           │                               │    │
│  │  │   或 医疗机构执业许可证  │                               │    │
│  │  │   / 备案凭证           │                               │    │
│  │  │                       │                               │    │
│  │  │   [点击上传]           │                               │    │
│  │  └───────────────────────┘                               │    │
│  │                                                          │    │
│  └──────────────────────────────────────────────────────────┘    │
│                                                                  │
│  *企业(单位)名称  [__________________________]  法定代表人 [______] │
│  *省市区  [省▼] [市▼] [区▼]                                       │
│  *详细地址  [______________________________________________]       │
└──────────────────────────────────────────────────────────────────┘
```

#### OCR 触发与填充规则

| 上传证照 | OCR 识别后自动填充字段 |
|---------|-------------------|
| 营业执照 | 企业(单位)名称、法定代表人、省市区、详细地址 |
| 医疗机构执业许可证 / 备案凭证 | 企业(单位)名称、法定代表人、省市区、详细地址 |

#### 识别状态与异常处理

- **识别中**：上传框内显示 loading 状态，字段区 skeleton 占位
- **识别成功**：字段自动填入，已识别字段显示蓝色边框（或右上角小标签），表示"来自证照识别，可手动修改"
- **图片模糊 / 识别失败**：图片缩略图下方红字提示「图片模糊，无法识别」，字段保持空白，用户手动填写
- **不上传**：用户可直接手动填写所有字段，与现有流程一致

### 2.2 仓库地址信息（收货地址）模块

#### 新增：身份证上传入口

位置：在「仓库地址信息」卡片顶部，收货人字段上方。

```
┌──────────────────────────────────────────────────────────────────┐
│ 仓库地址信息（收货地址）                                            │
│                                                                  │
│  ┌──────────────────────────────────────────────────────────┐    │
│  │  上传采购员身份证可自动识别填写（选填）                       │    │
│  │                                                          │    │
│  │  ┌──────────────┐  ┌──────────────┐                     │    │
│  │  │   人像面       │  │   国徽面       │                     │    │
│  │  │  [点击上传]   │  │  [点击上传]   │                     │    │
│  │  └──────────────┘  └──────────────┘                     │    │
│  └──────────────────────────────────────────────────────────┘    │
│                                                                  │
│  *收货人  [自动填入姓名]        *收货人手机号/电话  [默认注册手机号]  │
│  *采购员  [___________]        *采购员手机号  [___] ☑与收货人信息一致│
│  *所在地区  [省▼][市▼][区▼][街▼]        【同注册地址一致】          │
│  *详细地址  [________________]           【同注册地址一致】          │
│  *销售单打印地址  [____________]          【同注册地址一致】          │
└──────────────────────────────────────────────────────────────────┘
```

#### OCR 触发与填充规则

| 上传证照 | OCR 识别后自动填充字段 |
|---------|-------------------|
| 采购员身份证（人像面） | 收货人姓名 |

> 注：身份证 OCR 也会识别出身份证号，建议同步填充采购员身份证号字段（步骤2上传身份证图片时复用）。**待确认是否在此步骤一并填写。**

#### 收货人手机号默认值

- 默认填充用户注册时的手机号
- 用户可手动修改

#### 采购员信息快速填写

- 勾选「☑ 与收货人信息一致」→ 采购员姓名和手机号自动同步收货人信息
- 取消勾选 → 采购员字段清空，可单独填写

#### 识别状态与异常处理

- 同 2.1，图片模糊时在图片下方红字提示「图片模糊，无法识别」，字段保持空白

### 2.3 【同注册地址一致】快捷操作

**适用字段**：收货所在地区、收货详细地址、销售单打印地址（共 3 处）

**交互规则**：

| 场景 | 行为 |
|------|------|
| 点击【同注册地址一致】 | 从基本信息的省市区+详细地址复制填入对应字段 |
| 省市区四级下拉可正常映射 | 正常填入（省→市→区→街道全部带入） |
| 省市区四级下拉映射不上（如证照地址无街道层级） | 省市区县四个字段**全部清空**，用户手动填写 |
| 用户修改了注册地址后再点击 | 以最新填写的注册地址为准 |

**按钮样式**：灰色文字链，位于字段右侧，示例：

```
*所在地区  [省▼][市▼][区▼][街▼]  同注册地址一致
*详细地址  [________________________]  同注册地址一致
*销售单打印地址  [________________]  同注册地址一致
```

### 2.4 经营范围自动预选

**触发时机**：企业(单位)名称填写完成（失焦或 OCR 填入后）即触发

**逻辑优先级**：

```
Step 1：以企业名称调用主数据接口
         ↓ 有返回 → 按接口返回结果勾选
         ↓ 无返回
Step 2：按企业类型默认勾选（规则待补充）
```

**默认勾选规则（待补充）**：

| 企业类型 | 默认勾选项 |
|---------|---------|
| 单体药店 | 待补充 |
| 连锁总店 | 待补充 |
| 连锁特许经营 | 待补充 |
| 批发企业 | 待补充 |
| 非公立医疗机构-营利性 | 待补充 |
| 非公立医疗机构-非营利性 | 待补充 |
| 个体诊所-营利性 | 待补充 |
| 个体诊所-非营利性 | 待补充 |
| 公立医疗 | 待补充 |
| 生产企业 | 待补充 |

**用户可在自动勾选基础上继续修改，已勾选项视觉上有标记**
