# 注册页面交互优化方案：证照前置 + OCR 自动填充

> **日期**：2026-03-27
> **背景**：业务反馈注册环节太复杂，用户需要手动填写大量与证照重复的信息
> **核心思路**：根据用户选择的企业类型，提前上传对应证照，OCR 识别后自动填充表单字段

---

## 一、现有流程 vs 改进流程

### 现有流程（3步）

```
步骤1: 选企业类型 → 手动填写全部字段（基本信息+仓库地址）
步骤2: 上传企业资质证明
步骤3: 完成
```

**痛点**：步骤1和步骤2割裂，用户先手动填一遍信息，再上传证照（证照里其实已经包含了这些信息）。

### 改进流程（3步不变，步骤1内部重组）

```
步骤1:
  1.1 选企业类型（不变）
  1.2 ⭐ 新增：上传关键证照区（根据企业类型动态展示）
  1.3 基本信息（OCR 自动填充 + 人工校对）
  1.4 仓库地址信息（OCR 自动填充 + 人工校对）
步骤2: 上传剩余资质证明（发票开票资料、食品许可证等非填充类证照）
步骤3: 完成
```

---

## 二、步骤1 页面交互详细设计

### 2.1 用户类别选择（保持不变）

与现有页面一致，选择后触发下方证照上传区的动态变化。

### 2.2 ⭐ 新增：证照上传区

> 位置：在「用户类别」和「基本信息」之间，新增一个卡片区域。

#### 设计要点

- **标题**：「上传证照，自动填写」
- **副标题**：「上传以下证照，系统将自动识别并填充表单信息，识别有误可手动修改」
- **根据企业类型动态展示不同的上传卡片：**

| 企业类型 | 展示的上传卡片 | 自动填充的字段 |
|---------|-------------|-------------|
| **单体药店/连锁总店/批发企业/连锁特许经营** | ① 营业执照 ② 药品经营许可证 ③ 采购委托书 | 见下方映射表 |
| **非公立医疗机构-营利性 / 个体诊所-营利性** | ① 营业执照 ② 医疗机构执业许可证 ③ 采购委托书 | 见下方映射表 |
| **非公立医疗机构-非营利性 / 个体诊所-非营利性 / 公立医疗** | ① 医疗机构执业许可证 ② 采购委托书 | 见下方映射表 |
| **生产企业** | ① 营业执照 ② 采购委托书 | 见下方映射表 |

#### 上传卡片交互

```
┌─────────────────────────────────────────────────────────────────┐
│  上传证照，自动填写                                                │
│  上传以下证照，系统将自动识别并填充下方表单，识别有误可手动修改            │
│                                                                 │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐          │
│  │   📄          │  │   📄          │  │   📄          │          │
│  │  营业执照      │  │ 药品经营许可证  │  │  采购委托书    │          │
│  │              │  │              │  │              │          │
│  │ [点击上传]    │  │ [点击上传]    │  │ [点击上传]    │          │
│  │  推荐先传 ⭐  │  │              │  │              │          │
│  └──────────────┘  └──────────────┘  └──────────────┘          │
│                                                                 │
│  💡 暂时没有证照？可以跳过，直接手动填写下方表单                       │
└─────────────────────────────────────────────────────────────────┘
```

**交互细节：**
- 每张卡片支持拍照/相册上传/文件上传
- 上传后显示缩略图 + 识别状态（识别中... → 识别完成 ✅）
- 识别完成后，下方表单字段自动填入，并对已填充字段加视觉标记
- 可标注推荐上传顺序（如「推荐先传」），引导用户先传能填充最多字段的证照
- **可跳过**：用户可以不上传任何证照，直接手动填写（兼容现有流程）

### 2.3 基本信息区（改造后）

表单字段保持不变，但增加 OCR 自动填充能力：

```
┌─────────────────────────────────────────────────────────────────┐
│  基本信息                                                        │
│                                                                 │
│  *企业(单位)名称  ┌─────────────────────────┐   法定代表人  ┌───────────────┐  │
│                  │ XX大药房有限公司  🤖 已识别 │              │ 张三  🤖 已识别 │  │
│                  └─────────────────────────┘              └───────────────┘  │
│                                                                 │
│  *省市区  ┌──────┐ ┌──────┐ ┌──────┐                            │
│          │ 广东省 │ │ 深圳市 │ │ 南山区 │  🤖 已从药品经营许可证识别    │
│          └──────┘ └──────┘ └──────┘                            │
│                                                                 │
│  *详细地址  ┌──────────────────────────────────────┐             │
│           │ 科技园南路XX号  🤖 已识别               │             │
│           └──────────────────────────────────────┘             │
└─────────────────────────────────────────────────────────────────┘
```

**交互细节：**
- OCR 填充的字段右侧显示 `🤖 已识别` 标签（蓝色小标签），表明来源
- 悬浮/点击标签可显示「来自：营业执照」，告知用户数据来源于哪张证照
- 所有 OCR 填充的字段均可编辑，用户点击即可修改
- 未被 OCR 覆盖的字段保持空白，由用户手动填写
- 如果同一字段被多张证照识别（如企业名称同时出现在营业执照和药证上），取主证照的结果（见下方映射表），不一致时高亮提醒

### 2.4 仓库地址信息区（改造后）

```
┌─────────────────────────────────────────────────────────────────┐
│  仓库地址信息（收货地址）                                          │
│                                                                 │
│  *收货人  ┌──────────────┐    *收货人手机号/电话  ┌──────────────┐  │
│          │ 李四  🤖 已识别 │                      │              │  │
│          └──────────────┘                      └──────────────┘  │
│                                                                 │
│  *采购员  ┌──────────────┐    *采购员手机号/电话  ┌──────────────┐  │
│          │ 李四  🤖 已识别 │                      │              │  │  ☑ 与收货人信息一致
│          └──────────────┘                      └──────────────┘  │
│                                                                 │
│  *所在地区  （从基本信息的省市区自动带入） 🤖                         │
│                                                                 │
│  *详细地址  （从基本信息的详细地址自动带入） 🤖                       │
│                                                                 │
│  *销售单打印地址  （从注册地址自动拼接带入） 🤖                      │
│                                                                 │
│  ⚠️ 如收货地址与销售单打印地址不一致，请取消勾选并单独编辑            │
└─────────────────────────────────────────────────────────────────┘
```

**交互细节：**
- 采购委托书 OCR 识别 → 自动填入采购员姓名、收货人姓名
- 默认勾选「与收货人信息一致」，收货人字段从采购员自动带入
- 收货地址默认从基本信息的注册地址带入
- 销售单打印地址默认从注册地址自动拼接生成
- 用户可取消勾选后分别编辑

## 四、异常场景处理

| 场景 | 处理方式 |
|------|--------|
| OCR 识别失败 | 提示「识别失败，请手动填写」，字段保持空白可手动输入 |
| OCR 识别置信度低 | 字段标橙色边框 + 提示「请确认此项是否正确」 |
| 证照模糊/不完整 | 提示「图片不清晰，建议重新上传」，同时允许手动填写 |
| 企业类型与药证首字母不匹配 | 弹窗提醒「您选择的企业类型为XX，但药品经营许可证编号显示为YY类型，请确认」 |
| 营业执照与药证企业名称不一致 | 高亮提醒「两张证照的企业名称不一致，请确认」 |
| 用户不想上传证照 | 「跳过」按钮，回退到纯手动填写模式（兼容现有流程） |
| 用户中途切换企业类型 | 清空已上传证照和已填充字段，重新展示对应证照上传卡片 |

## 五、改进收益预估

| 指标 | 现状 | 预期 |
|------|------|------|
| 步骤1填写时间 | 手动填写约 10-15 分钟 | OCR 自动填充 + 校对约 3-5 分钟 |
| 填写错误率 | 较高（手动抄写易出错） | 大幅降低（OCR 填充 + 人工校对） |
| 步骤2重复操作 | 证照只在步骤2上传，步骤1信息与证照脱节 | 证照提前上传，步骤1和步骤2合并部分工作 |
| 退回率 | 因信息与证照不一致被退回 | 提前校验一致性，减少退回 |
