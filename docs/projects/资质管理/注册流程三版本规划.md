# 注册流程三版本规划

> **整理日期**：2026-04-02

---

## 核心目标

让用户尽可能少填字段、少上传图片，最快完成注册并具备下单资格。

## 三阶段权限说明

| 权限 | 触发条件 |
|------|---------|
| 可浏览商品/看价格 | BD 完成初审 |
| 可下单 | 补充采购/收货信息 |
| 可发货 | 完成证照上传并通过审核 |

---

## 版本一：线上现有版本

### 第一步：基本信息（→ 可浏览）

```
用户输入企业名称
    ↓
调用主数据接口（企查查等数据源）
    ├── 命中 → 自动带出注册地址、法定代表人、经营范围
    └── 未命中 → 用户手动填写全部字段
    ↓
用户提交审核 → BD 完成初审
    ↓
✅ 可浏览商品、看价格
```

**主要问题**：主数据对诊所客户覆盖率低，未命中时全靠手填，流失率高。

### 第二步：采购收货信息（→ 可下单）

用户手动填写以下全部字段：
- 采购员姓名 / 采购员手机号
- 收货员姓名 / 收货员手机号
- 收货地址（省市区街道 + 详细地址）
- 销售单打印地址

**主要问题**：字段多、全手填、与第一步信息重复，用户体验差。

### 第三步：证照上传（→ 可发货）

根据客户类型上传不同证照，全部手动上传，等待审核。

**主要问题**：全手动，无引导，审核状态不透明。

## 版本二：下周四版本（2026-04-10）

### 第一步：基本信息（改动）

```
用户输入企业名称
    ↓
调用主数据接口
    ├── 命中 → 自动带出注册地址、法定代表人、经营范围
    └── 未命中 → ⭐ 新增：上传营业执照 / 医疗机构执业许可证 / 备案凭证
                        ↓ OCR 识别
                        自动填充：企业名称、法定代表人、注册地址
                        识别失败 → 红字提示，手动填写
    ↓
用户提交审核 → BD 完成初审 → ✅ 可浏览
```

**改动价值**：解决诊所客户主数据覆盖率低的问题，不再依赖主数据命中。

### 第二步：采购收货信息（改动）

⭐ 新增：上传采购员身份证
- OCR 识别 → 自动填充采购员姓名、收货员姓名

⭐ 新增默认值：
- 收货员手机号 / 采购员手机号 → 默认填充注册手机号
- 收货地址 → 默认填充注册地址
- 销售单打印地址 → 默认填充注册地址

> **默认值选型依据**：现阶段新客主力为诊所/单体类客户，收货地址通常与注册地址一致。连锁和批发客户两者通常不同，但当前存量小，默认值可修改兜底。

用户校对后提交 → ✅ 可下单

### 第三步：证照上传（无改动）

证照需加盖公章，无法通过主数据获取，仍为手动上传。

## 版本三：终极版本

**核心思路**：如果用户已在其他平台（如药师帮、小药药等）完成过注册，其证照和注册信息已经过对方平台审核，可引导用户授权，由 AI 从竞品平台抓取全部信息，一键完成 1药城 的注册流程。

### 新增入口：竞品平台一键导入

```
注册页新增入口：「已在其他平台注册过？一键导入资料」
    ↓
用户选择平台（药师帮 / 小药药 / 药九九 / 其他）
    ↓
用户在对应平台完成授权（扫码登录或账号授权）
    ↓
AI 抓取该平台上的：
    - 企业基本信息（名称、法人、地址、统一社会信用代码）
    - 采购员 / 收货员信息
    - 证照图片（营业执照、药品经营许可证、医疗机构执业许可证等）
    ↓
自动填充 1药城 注册全部字段 + 自动上传证照
    ↓
用户确认信息无误 → 一键提交
    ↓
✅ 直接进入审核，审核通过即可发货
```

### 三步变一步

| 现有流程 | 终极版本 |
|---------|---------|
| 第一步：填基本信息 → BD初审 | |
| 第二步：填采购收货信息 | → 一键导入 → 用户确认 → 统一提交 |
| 第三步：上传证照 → 质管审核 | |
| **合计：3段流程，多次等待** | **合计：1段流程，1次等待** |

### 关键依赖与风险

| 项目 | 说明 |
|------|------|
| 用户授权 | 必须由用户主动授权，不可未经授权抓取 |
| 竞品平台反爬 | 药师帮等平台可能有反爬机制，需评估技术可行性 |
| 数据准确性 | 竞品平台数据可能与1药城要求格式不完全一致，需字段映射和人工校验 |
| 法律合规 | 需法务确认用户授权后爬取第三方平台数据的合规边界 |
| 证照时效性 | 竞品平台上的证照可能已过期，需 OCR 识别有效期并提示 |

### 近期可落地的子集

在完整方案落地前，可先做一个轻量版：

**「已有证照图片？直接上传包」**
- 用户一次性上传所有证照图片（不限格式、不限顺序）
- AI 自动识别每张图片是什么证照类型
- 自动分类填入对应字段（营业执照框、药品经营许可证框等）
- 识别有问题的高亮提示，用户确认

不依赖竞品平台授权，只要用户手机相册里有图，体验大幅提升。

## 三版本对比

| 维度 | 线上版本 | 下周四版本 | 终极版本 |
|------|---------|-----------|---------|
| 第一步最少操作 | 输入企业名称（命中主数据） | 上传1张证照 | 授权竞品账号 |
| 第二步最少操作 | 手填6个字段 | 上传身份证 + 确认默认值 | 无需操作 |
| 第三步最少操作 | 手动上传N张证照 | 手动上传N张证照 | 自动导入 |
| 全程最快完成 | 10-15分钟 | 3-5分钟 | <1分钟 |
| 对诊所客户友好度 | 低（主数据覆盖率差） | 中 | 高 |
| 技术复杂度 | — | 低（OCR+默认值） | 高（授权+爬取+映射） |
| 核心依赖 | 主数据覆盖率 | 无外部依赖 | 法务合规 + 反爬评估 |
