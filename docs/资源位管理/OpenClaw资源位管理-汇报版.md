## P1｜我理解的 OpenClaw（30s）

OpenClaw 不是聊天机器人，是**永远在线的运营助理**：

| 能力 | 一句话说明 |
|------|----------|
| **听得懂** | 截图企微聊天 → 自动提取品牌、时间、偏好 |
| **记得住** | 资源位流量基线、排期状态、团队偏好，全在知识库里 |
| **跑得快** | 一句话跨 5 个系统完成配置 |
| **会汇报** | 每天主动播报数据，异常自动预警 |

| ⑤ | 创建用户集 | CRM 手动创建条件组合 | 运营描述圈选条件，自动调 CRM API 创建 | 全自动 |
| ⑥ | 创建优惠券 | 后台手动填十余个字段 | 运营描述券规则，自动调 API 创建并关联用户集 | 全自动 |

| ⑨ | 收集活动商品 | 业务方提供 Excel | AI 按品牌+活动类型自动推荐商品列表 | AI辅助 |

| ⑮ | 数据监控 | 手动从神策拉数据 | 每日自动拉取，生成日报+异常预警 | 全自动 |

## P5｜实现方案：1 个 Skill + 11 个底座 API（60s）

### Skill 架构

```
资源位全流程管理 Skill
├── 模块1：需求识别（截图识别 → 结构化提取）
├── 模块2：排期推荐（读取腾讯文档排期表 → 流量排序 → 推荐+话术）
├── 模块3：用户圈选（解析圈选条件 → 调 CRM API 创建用户集）
├── 模块4：配置执行（优惠券+落地页+入口 → 一键配置）
├── 模块5：审批上线（自动提交审批 → 监听结果 → 发布/通知）
└── 模块6：数据监控（日报播报 → 异常预警 → 效果总结）
```

### 需要技术封装的底座 API

| # | API | 说明 | 优先级 |
|---|-----|------|:-----:|
| 1 | 腾讯文档读写 API | 读取/更新排期表（OAuth 授权，工作量约 1-2 天） | P0 |
| 2 | 神策数据查询 API | 按资源位+时间维度查询点击、访问、转化数据 | P0 |
| 3 | CRM 用户集创建 API | 按条件组合创建用户集（客户属性、业务快照、关联商品等维度） | P1 |
| 4 | CRM 用户集查询 API | 查询已有用户集列表、客户数、条件详情 | P1 |
| 5 | 优惠券创建 API | 创建优惠券并关联用户集（名称、面额、门槛、有效期、限领规则等） | P1 |
| 6 | 入口配置 API | 创建/修改资源位入口（标题、图片、链接、时段、生效周期、企业类型） | P1 |
| 7 | maps 落地页 API | 创建落地页、上传头图、挂载优惠券、导入商品 | P1 |
| 8 | 审批流 API | 提交审批单到内部系统 + 查询/回调审批结果 | P1 |
| 9 | 商品查询 API | 按品牌/品类查询商品列表 | P1 |
| 10 | 图片上传 API | 上传封面图/头图到后台 | P2 |
| 11 | 企微消息推送 API | 推送日报/设计需求单/审批结果到企微群 | P2 |

## P8｜产品经理的价值：不是成本，是生产"成本节省"的人（90s）

**PM 不是被 AI 替代的人，PM 是拿 AI 去替代重复劳动的人。**

### 1. 定义"做什么"和"不做什么"

AI 能跑完 16 个环节，但**哪些该自动化、哪些必须留给人**，是 PM 的判断：
- 审批不能自动通过——合规红线不可逾越
- 设计出图不用 AI——品牌方对视觉有主观要求
- 这些边界判断，AI 没有立场，PM 有

### 2. 把散落的经验变成可执行的规则

运营的经验在脑子里、群聊里、Excel 里。**谁来定义知识库结构、谁来决定什么该沉淀什么该丢弃**，是 PM 的工作：
- "化学药可以勾选生化药品和抗生素制剂"——怎么变成 Skill 逻辑？
- "流量大的位置优先给 JBP 商家"——要不要写进推荐算法？
- AI 是执行规则的，PM 是**制定规则的**

### 3. 设计人和 AI 的协作边界

**什么时候让 AI 自己跑、什么时候必须停下来等人确认**，这个流程设计决定了系统的可靠性：
- 用户集创建完要让运营确认命中人数——防止圈选条件写错影响几千家客户
- 优惠券预算要设上限拦截——防止配置错误造成资损
- 这些"安全阀"的位置，是 PM 设计的

### 4. 跨团队的需求翻译和优先级决策

AI 不会参加排期会，不会判断"这周九芝堂和北京同仁堂谁优先级更高"：
- 业务价值的权衡、团队间利益的协调、资源冲突的裁决——这些是人的判断
- PM 是**业务和技术之间的翻译官**：把运营痛点翻译成 Skill，把技术边界翻译成产品方案

### 5. 持续量产 AI 能力，ROI 3-5 倍

```
裁掉 1 个 PM → 省 30 万/年
留下 1 个 PM → 资源位场景省 80 万/年，还能接着做下一个场景
```

**每个 PM 就是一台"AI 能力生产机"。AI 是花园不是雕塑——没人打理就荒了。**

### 一句话总结

> **AI 时代，PM 的价值不是"会画原型"，而是"能把业务问题变成 AI 方案，持续为公司量产降本增效的能力"。这份方案本身，就是最好的证明。**
