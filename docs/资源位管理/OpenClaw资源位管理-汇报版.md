---

## P1｜我理解的 OpenClaw（30s）

OpenClaw 不是聊天机器人，是**永远在线的运营助理**：

| 能力 | 一句话说明 |
|------|----------|
| **听得懂** | 截图企微聊天 → 自动提取品牌、时间、偏好 |
| **记得住** | 资源位流量基线、排期状态、团队偏好，全在知识库里 |
| **跑得快** | 一句话跨 5 个系统完成配置 |
| **会汇报** | 每天主动播报数据，异常自动预警 |

## P2｜痛点：94% 的时间在执行（60s）

**一次资源位配置 = 16 个环节 × 4 个角色 × 5 个系统**

```
运营一天 8 小时：

██████████████████░░  3.0h  搬运数据和配置     ← AI 可替代
████████░░░░░░░░░░░░  2.0h  群里沟通对齐需求   ← AI 可辅助
██████░░░░░░░░░░░░░░  1.5h  拉数据做报表       ← AI 可替代
████░░░░░░░░░░░░░░░░  1.0h  审批和检查         ← AI 可替代
██░░░░░░░░░░░░░░░░░░  0.5h  策略思考和优化     ← 这才是运营该做的事
```

**94% 执行，6% 思考。OpenClaw 要翻转这个比例。**

## P3｜16 个环节逐一拆解（60s）

| # | 环节 | 当前做法 | OpenClaw 之后 | 自动化 |
|---|------|--------|-------------|:-----:|
| ① | 业务方提需求 | 企微群聊沟通 | 截图识别 → 自动提取品牌、时间、偏好 | 全自动 |
| ② | 查排期+推荐资源位 | 看腾讯文档，凭经验推荐 | 自动读取排期表 + 按流量排序推荐 + 生成回复话术 | 全自动 |
| ③ | 锁定排期 | 手动更新腾讯文档 | 运营确认后自动更新排期表 | 全自动 |
| ④ | 沟通活动细节 | 企微反复沟通 | AI 生成确认清单，运营一次性发给业务方确认 | AI辅助 |
| ⑤ | 创建用户集 | CRM 手动创建条件组合 | 运营描述圈选条件，自动调 CRM API 创建 | 全自动 |
| ⑥ | 创建优惠券 | 后台手动填十余个字段 | 运营描述券规则，自动调 API 创建并关联用户集 | 全自动 |
| ⑦ | 提设计需求 | 在线表格提需求 | 自动生成设计需求单（品牌、主题、尺寸、交付时间） | 全自动 |
| ⑧ | 设计师出图 | 设计师排期制作 | AI 不介入，等待设计师交付 | 人工 |
| ⑨ | 收集活动商品 | 业务方提供 Excel | AI 按品牌+活动类型自动推荐商品列表 | AI辅助 |
| ⑩ | 配置落地页 | maps 手动搭建 | 自动调 maps API：上传头图、挂载券、导入商品 | 全自动 |
| ⑪ | 配置入口 | 后台逐个填字段 | 自动调 API 填充所有字段 | 全自动 |
| ⑫ | 上线前检查 | 人工逐项核查 | 自动校验图片、链接、时段、库存 | 全自动 |
| ⑬ | 提交审批 | 人工提交 | 自动提交审批单，附配置摘要+预览链接 | 全自动 |
| ⑭ | 上线 | 审批通过后手动发布 | 审批通过自动发布；驳回自动通知并提示修改项 | 全自动 |
| ⑮ | 数据监控 | 手动从神策拉数据 | 每日自动拉取，生成日报+异常预警 | 全自动 |
| ⑯ | 效果反馈 | 基本没做 | 自动生成效果报告+优化建议，推送给业务方 | 全自动 |

**16 个环节：12 个全自动，2 个 AI 辅助，2 个人工（设计出图 + 审批）。**

## P5｜实现方案：1 个 Skill + 11 个底座 API（60s）

### Skill 架构

```
资源位全流程管理 Skill
├── 模块1：需求识别（截图识别 → 结构化提取）
├── 模块2：排期推荐（读取腾讯文档排期表 → 流量排序 → 推荐+话术）
├── 模块3：用户圈选（解析圈选条件 → 调 CRM API 创建用户集）
├── 模块4：配置执行（优惠券+落地页+入口 → 一键配置）
├── 模块5：审批上线（自动提交审批 → 监听结果 → 发布/通知）
└── 模块6：数据监控（日报播报 → 异常预警 → 效果总结）
```

### 需要技术封装的底座 API

| # | API | 说明 | 优先级 |
|---|-----|------|:-----:|
| 1 | 腾讯文档读写 API | 读取/更新排期表（OAuth 授权，工作量约 1-2 天） | P0 |
| 2 | 神策数据查询 API | 按资源位+时间维度查询点击、访问、转化数据 | P0 |
| 3 | CRM 用户集创建 API | 按条件组合创建用户集（客户属性、业务快照、关联商品等维度） | P1 |
| 4 | CRM 用户集查询 API | 查询已有用户集列表、客户数、条件详情 | P1 |
| 5 | 优惠券创建 API | 创建优惠券并关联用户集（名称、面额、门槛、有效期、限领规则等） | P1 |
| 6 | 入口配置 API | 创建/修改资源位入口（标题、图片、链接、时段、生效周期、企业类型） | P1 |
| 7 | maps 落地页 API | 创建落地页、上传头图、挂载优惠券、导入商品 | P1 |
| 8 | 审批流 API | 提交审批单到内部系统 + 查询/回调审批结果 | P1 |
| 9 | 商品查询 API | 按品牌/品类查询商品列表 | P1 |
| 10 | 图片上传 API | 上传封面图/头图到后台 | P2 |
| 11 | 企微消息推送 API | 推送日报/设计需求单/审批结果到企微群 | P2 |

## P8｜产品经理的价值：不是成本，是生产"成本节省"的人（90s）

### 先算一笔账

```
裁掉 1 个 PM：
  省下 ≈ 30 万/年

留下 1 个 PM：
  资源位场景：释放运营团队 80% 执行时间 ≈ 等效节省 80 万/年
  做完资源位，接着做资质审核、商品对码、客户注册……
  每多做一个场景，就多省一份成本

  PM 的 ROI = 产出的降本金额 ÷ PM 自身成本 ≈ 3-5 倍
```

**PM 不是被 AI 替代的人，PM 是拿 AI 去替代重复劳动的人。**

### 裁掉 PM 之后会发生什么

| 时间 | 会发生的事 | 后果 |
|------|----------|------|
| 第 2 个月 | 药监政策变了，排期规则要调整 | 没人改 Skill，运营绕过 AI 手动操作 |
| 第 3 个月 | 公司新增一种资源位类型 | Skill 不认识，运营回到手工模式 |
| 第 6 个月 | 其他部门想复制这套方案 | 没人能设计新的 Skill，机会白白浪费 |
| 第 12 个月 | 知识库过时，AI 推荐越来越不准 | 运营彻底放弃使用，9 周建设全部归零 |

**AI 是花园，不是雕塑。雕塑造完可以走人，花园没人打理就荒了。**

### PM 是公司里唯一能"量产 AI 能力"的角色

```
技术 → 能封装 API，但不知道该封装哪个、优先级怎么排
运营 → 知道痛点，但不知道怎么变成 Skill
老板 → 知道要降本增效，但不知道哪个场景 ROI 最高

PM → 唯一能把"业务痛点"翻译成"可落地的 AI 方案"的人
```

资源位做完了，下一个做什么？

| 场景 | 预估节省 | PM 工作 |
|------|---------|--------|
| 资源位全流程智能化 | 80 万/年 | ✅ 本次方案 |
| 资质审核自动化 | 待评估 | 下一个 Skill |
| 商品对码智能化 | 待评估 | 下一个 Skill |
| 客户注册自助化 | 待评估 | 下一个 Skill |

**每个 PM 就是一台"AI 能力生产机"。留 1 个 PM，持续产出降本方案；裁 1 个 PM，所有在建和待建的 AI 场景全部停滞。**

### 一句话总结

> **AI 时代，PM 的价值不是"会画原型"，而是"能把业务问题变成 AI 方案，持续为公司量产降本增效的能力"。这份方案本身，就是最好的证明。**
